## Overview

Textarea is the multi-line variant of Input. It shares the same token system and states but grows vertically to accommodate longer text. 12 variants total (2 sizes × 6 states).

## When to use

- Descriptions, notes, or messages (multi-line)
- Any text input where the user may write more than one sentence

## When not to use

- Short, single-line inputs — use [Input](/docs/components/input) instead

## Sizes

| Size | Height | PaddingH | PaddingV |
|------|--------|----------|----------|
| `sm` | 80px | 12px (spacing/4) | 8px (spacing/3) |
| `md` | 120px | 16px (spacing/5) | 12px (spacing/4) |

## States

Same 6 states as Input: default, hover, focus, error, disabled, readonly.

The focus state shows the teal focus ring (DROP_SHADOW, spread 3px, `focus/Ring` colour).

## Accessibility

- Include a `<label>` linked via `for`/`id`
- Set `rows` attribute to hint expected content length
- Consider `maxlength` + a character counter for limited inputs
- Avoid disabling resize unless layout requires it
